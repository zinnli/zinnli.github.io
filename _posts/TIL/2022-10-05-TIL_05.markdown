## React TodoList

\*\* TodoList에 대한 자세한 내용은 내일 따로 작성(링크 추가 예정)<br/>
<br/>
내가 진행한 순서만 정리해보면<br/>

1. 과제의 요구사항을 바탕으로 `전체 Layout` 작성
2. `html 의 내용`이 입력되는 부분을 우선적으로 작성
3. 기능단위로 `component 분리` + `틀이 되는 css` 작성
4. list.jsx와 form.jsx 가 서로에게 데이터를 줄 수 있도록 `props 전달`
5. form.jsx에 해당하는 내용부터 우선 진행<br/>(제목/내용 입력 후 `추가하기 버튼` 클릭 -> 리스트에 `업데이트`)
6. 입력 확인 후 list.jsx에서 각 `입력내용 확인`
7. todo.jsx에 수정(완료/취소), 삭제 버튼 구현
8. `삭제 버튼` 기능 먼저 우선 구현
9. `수정` 시 버튼에 따라 위치 변경 / `버튼 value` 변경(완료/취소)
10. 전체적인 내용 정리 후` css 최종 수정`
    <br/><br/>

## git 삭제 파일 복구

예전에 미니 프로젝트를 하다가 거하게 날린 사람으로,, 같은 조원분이 반나절 이상 작업했다던 파일을 날리고 속상해 하는 모습을 보니.. 그냥 둘 수가 없었다.<br/>
구글링 열심히 해본 결과 어떤 분의 게시물을 보게 되었고 조원분을 설득해서 해당 방법을 시도해본 결과 2번정도 오류나고 3번째에 성공했다.<br/>

-- 조원분의 경우, `잘못된 폴더에서 git add를 시도`했고 오류를 겪다가 다른 분의 Pull을 해보라는 조언에 작업한 깃 폴더에서 Pull을 진행했고 그동안 진행한 파일을 다 날리는 결과가 생겼다.<br/><br/>

블로그에 따르면<br/>
`Commit 하지 않은 파일`은 해당하는 폴더에서<br/>

```
1. git fsck --lost-found
```

git add 단계에서 삭제한 파일은 blob으로 해서 각 파일들이 생성되고<br/>
디렉토리도 삭제하는 경우 tree로 끝나는 단어?가 생김<br/>
-> 숫자로 시작한 10-15개 이상의 단어가 생긴다<br/><br/>
-> 조원분의 경우 blob으로 끝나는 내용이 표시되었다

```
2. git show 복사한id
```

(이거는 시도 안해봤다,,ㅎㅎ)

```
3. git show 복사한id > 새 파일 이름

git show efsef32r2424323523234 > hello.txt
```

위처럼 파일에 저장을 해주면 txt 파일에 삭제한 파일의 내용이 입력되는데 txt 파일을 보면 한글은 깨진 글꼴로 뜨게 되는데 해당 부분은 어쩔 수 없이 수기로 수정을 해줘야하는듯...
